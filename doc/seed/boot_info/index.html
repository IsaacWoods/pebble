Module seed::boot_info

source ·
Expand description

The “boot info” refers to a data structure passed from Seed to the kernel, telling it about the platform it’s running on, memory it can use, and about other objects Seed has been asked to load into memory.

Seed implementations generally don’t have their own heaps, and so these data structures need to be representable without heap allocation. For this reason, the heapless crate is used to supply stack-backed containers - the resulting data structure is then serialized using ptah, and can then be deserialized in the kernel.
